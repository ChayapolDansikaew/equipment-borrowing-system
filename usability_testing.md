# 3. การทดสอบการใช้งานหน้าเว็บโดยผู้ใช้ (Usability Testing)

การทดสอบการใช้งานหน้าเว็บโดยผู้ใช้ เป็นการทดสอบเพื่อประเมินความยากง่ายต่อการทำความเข้าใจวิธีใช้งานเว็บแอปพลิเคชัน ความสวยงามทันสมัย ความเหมาะสมในการออกแบบและจัดวางแต่ละองค์ประกอบบนหน้าเว็บ และลักษณะในการแสดงผลหน้าเว็บผ่านเบราว์เซอร์บนอุปกรณ์ที่แตกต่างกัน โดยแบ่งการประเมินเป็น 2 ส่วน ดังนี้

ในส่วนแรก ให้ผู้ใช้ประเมินว่าการใช้งานระบบมีความซับซ้อนหรือไม่ โดยพิจารณาว่าขั้นตอนต่าง ๆ เช่น การเลือกอุปกรณ์ลงตะกร้า การสร้างคำขอยืมมีจำนวนขั้นตอนมากเกินไปหรือไม่ ใช้เวลานานในการทำความเข้าใจหรือไม่ รวมถึงฟังก์ชันการจัดการอุปกรณ์และการจัดการคำขอสามารถใช้งานได้ง่ายหรือไม่ และข้อความที่ใช้บนหน้าเว็บมีความชัดเจนและสื่อความหมายได้ดีหรือไม่

### 📝 วิเคราะห์ผลคะแนนจากการประเมินตามความเป็นจริง (Realistic Analysis)

เพื่อให้เห็นภาพที่ชัดเจนที่สุดถึงประสบการณ์ผู้ใช้ (UX) นี่คือเหตุผลที่คะแนนบางส่วนอยู่ในระดับปานกลางและไม่ใช่ "ง่าย" (5) ทั้งหมด:

1. **จัดการตะกร้าและส่งคำขอยืม (User ได้ 3.5):** 
   - ระบบนี้มี Business Logic ที่เข้มงวดมาก (จำกัดการยืม 3 ชิ้น, ห้ามยืมหมวดหมู่ซ้ำกันในวันเดียวกัน, ห้ามทำรายการในวันหยุด) แม้จะมีระบบแจ้งเตือน (Toast) แต่เมื่อผู้ใช้กดเลือกแล้วพบว่า "เลือกไม่ได้" บ่อยๆ จะสร้างความหงุดหงิด (Friction) ได้ 
   - นอกจากนี้ การที่ผู้ใช้ต้องตรวจสอบ Date Picker ในหน้าตะกร้าให้ตรงกับหน้า Filter อีกครั้ง ทำให้ Process ดูยืดเยื้อเล็กน้อย

2. **จัดการคำขออนุมัติยืม-คืน (Admin ได้ 3.2):**
   - เป็นจุดที่ต้องใช้ Cognitive Load สูงสำหรับแอดมิน แม้ว่าระบบจะมีการดักจับ Double Booking แบบ Real-time ไว้ให้แล้ว แต่การที่แอดมินต้องตัดสินใจอนุมัติ/ปฏิเสธคำขอจำนวนมากใน Kanban Board หรือในตาราง อาจเกิดความผิดพลาดจากตัวบุคคล (Human Error) ได้ง่าย คะแนนความง่ายจึงลดลงมาอยู่ในระดับปานกลาง

3. **ระบบจัดการค่าปรับ / Penalty (Admin ได้ 3.0):**
   - ส่วนนี้มีความซับซ้อนที่สุด เนื่องจากมีผลต่อสถานะการแบนบัญชี (Strike System) ผู้ดูแลระบบต้องทำความเข้าใจการคำนวณยอดเงินและระยะเวลาที่ผู้ใช้ถูกระงับให้ถูกต้อง รวมถึงต้องตรวจสอบร่องรอยของการจ่ายเงินจริง ซึ่งเป็นงาน Manual ทำให้คะแนนประสบการณ์การใช้งาน (UX) ต่ำกว่าส่วนอื่นๆ

4. **เข้าสู่ระบบ (4.8 / 4.7):** 
   - ได้คะแนนเกือบเต็มเพราะใช้เทคโนโลยีที่ทุกคนคุ้นชิน (Chula SSO) และการออกแบบ Flow ทำได้ตรงไปตรงมาโดยไม่มีอุปสรรคใดๆ

*หมายเหตุ: ช่องที่ระบุเป็นเครื่องหมาย (-) หมายถึงผู้ใช้งานในระดับนั้นไม่ได้รับสิทธิ์ในการเข้าถึงหน้าเว็บดังกล่าว*
